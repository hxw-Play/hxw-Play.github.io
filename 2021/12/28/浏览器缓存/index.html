浏览器的缓存

浏览器的强缓存和协商缓存

浏览器的缓存(html,css,js,png,jpg………)

  • 协商缓存会和服务器进行一次交互(304)

    • 需要设置不能够强缓存 ctx.set(‘Cache-Control’, ‘no-cache’)
    • 有变化就会直接请求到最新的资源

  • Memory Cache：基于内存的缓存，读取高效速度快，但是一旦关闭网页，内存就释放了。

  • Disk Cache：基于磁盘的缓存，容量大，读取慢

  • 强缓存设置的时间过期 ——-> 协商缓存 ——> 服务器

    • 缓存设置的时间不过期 一直都会是老数据

基于 koa 实现的缓存策略

  • npm install koa2-generator -g
  • koa2 项目名称
